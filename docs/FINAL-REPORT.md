## ٧) يلزم المالك لاحقاً

1. **مفتاح Anthropic حقيقي** (`ANTHROPIC_API_KEY` في deploy.env) → `LLM_ENGINE=claude` (النموذج claude-sonnet-4-5) — حالياً mock بعينات سريرية ثابتة.
2. **مفاتيح مزود الدفع** (Moyasar أو Tap) — يُختبر مزود واحد بعملية حقيقية كاملة قبل قفله (DOC-09 §٣)؛ حالياً mock يسوّي فوراً.
3. **وجهة رفع مستشفى حقيقية** (endpoint FHIR + مفتاح في W-107) → `INTEGRATION_ENGINE=http` — حالياً mock يؤكد الرفع.
4. **خادم Oracle + deploy.env** (والمنطقة السعودية جدة/الرياض للإنتاج — شرط PDPL) وفتح 80/443/8080 من لوحة OCI إن لم يكن oci CLI مهيأ.
5. **دومين + DNS** يشير للخادم → TLS تلقائي عبر Caddy.
6. **ترخيص أنظمة الترميز** (ICD-10-AM/ACHI — IHACPA، SBS، SFDA GTIN) بإصداراتها الرسمية.
7. **عينة تسجيلات STT** (30+ تسجيلاً حقيقياً بموافقات) لقفل نموذج P1 وقياس خط أساس الدقة (DOC-17) — ثم `STT_ENGINE=whisper` مع `INSTALL_WHISPER=true`.
8. **خدمة مزامنة مرضى حقيقية** من نظام المستشفى — حالياً seed يحاكيها (لا إنشاء مرضى داخل Medify بقرار المالك).
9. جعل المستودع **خاصاً** إن أراد (البرومت طلب خاصاً والريبو الموفر عام — D-01).
10. بريد SMTP حقيقي للإشعارات الحرجة واستعادة كلمة المرور (`EMAIL_ENGINE=smtp`) — حالياً mock يكتب لصندوق صادر محلي.
